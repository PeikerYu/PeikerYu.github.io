我的第一个 GitHub Pages 博客是如何搭建的

一、工具准备：从零开始

为了搭建这个 Hexo 博客，我首先安装了三个关键工具：

  1. Git： 用于版本控制，并连接 GitHub。
  2. Node.js (LTS)： 运行 Hexo 框架的 Javascript 环境。
  3. Hexo-CLI： 通过 npm install -g hexo-cli 命令安装，用于创建和管理 Hexo 博客。

二、解决网络和权限问题（Clash 代理设置）

在初始化 Hexo (hexo init) 时，我遇到了连接 GitHub 失败的错误。这是因为 Git 需要通过代理访问国外服务器。

解决方法：

  1. 开启 Clash for Windows，并确保节点运行正常，找到 HTTP 端口 7890。
  2. 以管理员身份 运行 CMD。

四、下一步：配置部署信息

现在我需要配置 GitHub Pages 部署信息，将网站发布到互联网：

  1. 安装部署插件：npm install hexo-deployer-git --save
  2. 修改 _config.yml 配置文件底部的 deploy 部分，指向我的 GitHub 仓库 我的用户名.github.io。

接着，就是运行 hexo deploy 正式发布我的第一篇文章！
